Overview & Architecture

The AI Growth Engine is a complete outbound sales automation system built into the InstantSEOScan admin panel. It is exclusively accessible to users with the "admin" role and is completely invisible to regular users.

The system automates the entire outreach pipeline:

  1. Discover businesses with poor SEO using keyword + location searches
  2. Store discovered domains in the Leads CRM database
  3. Enrich leads with contact information (email, phone)
  4. Generate personalized outreach emails using templates with dynamic variables
  5. Send bulk email campaigns via the Resend API
  6. Track open rates, click rates, and replies
  7. Automate the entire workflow on a schedule (hourly / daily / weekly)
📌 Note: All Growth Engine API endpoints require a valid admin JWT token. Any attempt to access them without admin credentials returns HTTP 403 Forbidden.

Accessing the Growth Engine

The Growth Engine is embedded inside the Admin Panel — it is not a public-facing page and does not appear in the navigation for regular users.

  1. Log in with your Admin account at /login
  2. Navigate to /admin — you will be redirected if you are not an admin
  3. In the left sidebar, scroll to the "Growth Engine" section
  4. The section contains 5 tabs: Lead Scraper, Leads CRM, Campaigns, Email Templates, Automation
⚠️ Admin Only: The route /admin checks user.role === "admin" in the frontend. The API endpoints check the role server-side via JWT. Non-admin users are hard-blocked at both layers.
💡 Tip: The old standalone /outreach route now automatically redirects to /admin. You don't need to update any bookmarks.

Lead Scraper Engine

The Lead Scraper discovers potential clients by generating domain name patterns based on your keyword and city inputs, simulating how businesses in a niche are typically named online.

🎯 How to Run a Scrape

  1. Click "Lead Scraper" in the sidebar
  2. Enter an Industry / Keyword (e.g. "dentist", "plumber", "marketing agency")
  3. Optionally enter a City / Location (e.g. "New York")
  4. Select a Source: Google, Bing, Google Maps, Yellow Pages, or LinkedIn
  5. Set Max Leads (1–50 per run)
  6. Click "Start Scraping"
  7. Leads are saved to the Leads CRM automatically

📊 What Gets Scraped

Each discovered lead includes:

FieldDetails
Website domainAuto-generated from keyword+location patterns
Company NameDerived from domain slug
SEO ScoreRandom 10–65 (intentionally low — these are opportunities)
Issues3 random SEO issues from a standard checklist
StatusAlways starts as "new"
SourceThe selected search source
TagsKeyword + location
💡 Quick-Start Patterns: Click any example query button (e.g. "dentist in New York") to pre-fill the form for fast experimentation.
🔮 Production Upgrade Path: To scrape real Google/Maps results, integrate the SerpAPI (SERP_API_KEY) or ScraperAPI. The scraper endpoint at /api/admin/growth/scraper is designed to be augmented with real HTTP scraping logic without changing the frontend.

Leads CRM

The Leads CRM is your central database of all discovered leads. Each lead represents a business that has been identified as having poor SEO and is a potential client.

📋 Lead Statuses

StatusMeaning
newJust discovered, no contact attempted
contactedAn outreach email has been sent
repliedLead replied to your email
convertedLead became a paying client
ignoredLead dismissed / not a good fit

🔍 Filtering & Search

  1. Use the search bar to filter leads by company name or domain
  2. Leads with email addresses (ready for campaigns) are shown in the footer count
  3. Click "Launch Campaign" in the footer to go straight to the Campaigns tab

🗑️ Deleting Leads

Hover over any lead row and click the red trash icon to permanently delete it. This also removes all associated email logs and SEO reports.

⚠️ Email Requirement: Only leads with an email address can receive campaign emails. To add emails, either: (1) run the Auto Lead Enrichment module, or (2) manually create a lead via the API: POST /api/admin/growth/leads with an email field.

Email Templates

Templates define the structure of outreach emails. They support dynamic variables that are personalized per lead at send time.

✏️ Creating a Template

  1. Click "Email Templates" in the sidebar
  2. Click "New Template" button (top-right)
  3. Enter a name, subject line, and email body
  4. Use the personalization variables below in both subject and body
  5. Click "Save Template"

🔤 Personalization Variables

VariableReplaced With
{{companyName}}The lead's company name (e.g. "Acme Dental")
{{website}}The lead's domain (e.g. "acmedental.com")
{{seoScore}}The lead's SEO score (e.g. "42")

📝 Example Template

Subject: Quick SEO opportunity for {{companyName}}

Hello {{companyName}} team,

I ran a quick SEO check on {{website}} and found some opportunities
that could help improve your Google rankings.

Your current SEO score is {{seoScore}}/100.

Some of the issues I noticed:
- Missing meta descriptions
- Slow page speed
- Weak internal linking

I've put together a free detailed SEO report for your website.
Would you like me to send it over?

Best regards,
The InstantSEOScan Team
https://instantseoscan.com
💡 Best Practice: Keep templates concise (150–250 words). Be specific about the lead's website and SEO issues. Mention the free report to increase reply rates.

Email Campaigns

A Campaign bundles a template with a target audience (your leads). When you send a campaign, every qualifying lead receives a personalized version of the template email.

🚀 Creating & Sending a Campaign

  1. Click "Campaigns" in the sidebar
  2. Click "New Campaign" (top-right)
  3. Enter a campaign name and select an email template
  4. Click "Create Campaign" — the campaign starts in Draft status
  5. Click "Send Now" on the campaign card to dispatch emails
  6. Emails are sent to all leads with status "new" that have an email address
  7. After sending, lead statuses update to "contacted"

📊 Campaign Statuses

StatusMeaning
draftCreated but not yet sent
runningEmails have been dispatched (at least once)
completedCampaign fully finished
pausedManually paused mid-run
📌 Simulated vs. Live Sending: If EMAIL_API_KEY is not set in your Vercel environment variables, campaigns run in "simulated" mode — all email logs are still recorded in the database but no real emails are sent. This is useful for testing.
⚠️ Anti-Spam Best Practice: Do not send to more than 50–100 cold leads per day per sending domain. Warm up your sending IP/domain before large-scale campaigns. Use a dedicated subdomain like outreach@mail.instantseoscan.com.

Automation Rules

The Automation engine runs the Growth Engine modules on a schedule. Each module can be independently enabled/disabled and given its own schedule interval.

ModuleWhat It Does
Auto Lead DiscoveryRuns the scraper with saved keyword/location presets to discover new leads automatically
Auto SEO ScanningRuns SEO audits on newly discovered leads and scores them 0–100
Auto Lead EnrichmentScrapes contact pages of each lead to extract email addresses and phone numbers
Auto Email OutreachSends outreach emails to qualified leads (SEO score < 65) using active campaigns

⚙️ Enabling a Module

  1. Click "Automation" in the sidebar
  2. Find the module you want to enable
  3. Set the schedule: Hourly, Daily, or Weekly
  4. Click "Enable" — the module turns active (green)
  5. To pause, click "Disable"
📌 Cron Job Integration: Automation status is stored in the database. To actually execute the modules on schedule, set up a cron job (via Vercel Cron Jobs, n8n, or GitHub Actions) that calls POST /api/admin/growth/automation/run with a valid admin token. See Section 9 for example cron configuration.

Email Delivery Setup (Resend API)

InstantSEOScan uses Resend as the email delivery provider. Resend is the recommended provider for Vercel deployments due to its edge-compatible API.

🔑 Getting Your Resend API Key

  1. Go to resend.com and create a free account
  2. Add and verify your sending domain (e.g. mail.instantseoscan.com)
  3. Go to API Keys → Create API Key → copy the key
  4. Add it to Vercel: Dashboard → Your Project → Settings → Environment Variables
  5. Add two variables: EMAIL_API_KEY = your Resend key, and EMAIL_FROM = admin@instantseoscan.com
  6. Redeploy from Vercel dashboard

🧪 Testing Email Delivery

  1. Create a test lead via the Leads CRM with your own email address
  2. Create a template and a campaign
  3. Click "Send Now" — check your inbox for the personalized email
💡 Free Tier: Resend Free plan allows 100 emails/day and 3,000/month. For scale, upgrade to their Pro plan ($20/mo for 50,000 emails).

🕐 Cron Job Example (n8n / GitHub Actions)

# GitHub Actions: .github/workflows/growth-cron.yml
name: Growth Engine Cron
on:
  schedule:
    - cron: '0 8 * * *'  # Daily at 8am UTC
jobs:
  run-automation:
    runs-on: ubuntu-latest
    steps:
      - name: Trigger Automation
        run: |
          curl -X POST https://instantseoscan.com/api/admin/growth/scraper \
            -H "Authorization: Bearer ${{ secrets.ADMIN_JWT_TOKEN }}" \
            -H "Content-Type: application/json" \
            -d '{"keyword":"dentist","location":"New York","maxLeads":50}'

Security & Access Control

The Growth Engine is protected at three layers:

  1. Frontend Route Guard: /admin checks user?.role === "admin" — non-admin users are redirected to home
  2. API Auth Middleware: All /api/admin/growth/* endpoints verify the JWT via withAuth() middleware
  3. Role Check: After token verification, each endpoint checks user.role !== "admin" and returns 403 Forbidden if not admin
⚠️ Token Security: Never share your admin JWT token. Tokens expire after 7 days. If compromised, update JWT_SECRET in Vercel env vars — this invalidates all existing tokens.

Troubleshooting

IssueSolution
Scraper returns 0 leadsCheck if the keyword is too specific. Try broader terms like "dentist" instead of "family dentist New York city downtown".
Emails show as "simulated"Set EMAIL_API_KEY in Vercel env vars and redeploy. The Resend API key must be valid.
Campaign says "No leads with email"Leads need email addresses. Add them manually or run Auto Lead Enrichment to extract emails from websites.
403 on API callsYour session token may have expired. Log out and log back in as admin.
Template variables not replacedUse exact variable syntax: {"{{companyName}}"} — double curly braces, camelCase, no spaces.
Automation modules don't runEnabling a module only flags it in the DB. You need a cron job to call the API on schedule. See Section 9.
Database errors on saveCheck DATABASE_URL in Vercel env vars. Run npx prisma migrate deploy if tables are missing.
